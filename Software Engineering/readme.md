## Reflection
 
This project reinforced that software engineering is as much about communication as it is about code. The SRS, SDD, and STD exist so that developers, testers, and clients are all reading from the same page — and that only works if the documentation is precise and consistent.
 
Working on a real client system (KADA) made the work feel purposeful. Every use case we documented represents a real person — a cooperative member trying to apply for a loan, or a clerk trying to verify an application without needing to be physically present. That context made it easier to care about getting the details right.
 
If I were to redo this, I would propose establishing a shared writing standard at the start of the sprint so that section formatting and terminology stay consistent from day one, rather than needing to be reconciled at the end.
