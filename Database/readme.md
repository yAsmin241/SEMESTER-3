# 📚 Self-Reflection — SECP2523 Database (WBL)

## Reflection
 
Working with a real client added a layer of responsibility that assignments alone cannot replicate. I learned that a database is not just about tables and keys — it reflects real processes, real people, and real consequences if something goes wrong (like a loan left with only one guarantor). Proposing the guarantor replacement module taught me to look at a system critically and think about edge cases that users actually face.
 
If I were to redo this, I would engage the client earlier and ask more targeted questions upfront to reduce the back-and-forth during design. I would also explore using more specific data types (e.g. `DATE` instead of `VARCHAR` for date fields) to build a more robust schema from the start.
